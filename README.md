# 手术示教软件 Phase 1 PoC

这是阶段 1 开发用的 Windows 桌面 PoC 工程，用于验证需求文档中的核心前置能力：

- 4 路 USB/UVC 视频输入预览。
- 无采集卡时使用 4 路模拟源进行功能验证。
- 选择部分通道或全部通道录制。
- 通过板载声卡或 USB 麦克风采集音频。
- 录像文件本地存储、索引、基础回放、定位和删除。

## 技术边界

本工程采用 Electron + React + MediaRecorder 实现，适合阶段 1 快速验证 USB 采集卡是否以标准摄像头设备暴露、四画面预览是否稳定、选择性录制流程是否成立。

它不是最终医疗级采集内核。若 PoC 发现浏览器媒体栈在长时间录制、多路高分辨率、特定采集卡驱动或低延迟场景下不稳定，生产版应切换或补充 Windows 原生采集层，例如 Media Foundation、DirectShow、厂商 SDK 或 FFmpeg/GStreamer 适配层。

## 运行

```powershell
cd D:\CodeX\UST\phase1-poc
npm install
npm run dev
```

应用启动后先点击 `授权并刷新设备`。如果没有 USB 采集卡，可保持通道为 `模拟源`，直接验证预览、录制、索引和回放流程。

## 阶段 1 验证口径

| 验证项 | 通过标准 |
|---|---|
| 4 路预览 | 4 个通道可同时启动，真实设备或模拟源画面稳定显示。 |
| 选择性录制 | 可选择任意通道录制，停止后生成可回放文件和索引。 |
| 音频采集 | 可选择麦克风作为录制音频输入。 |
| 本地索引 | 录像记录包含通道、时间、文件名、大小和本地路径。 |
| 基础回放 | 可在应用内选择录像并播放。 |

## 已知限制

- 录制格式由 Chromium MediaRecorder 决定，通常为 WebM。
- 音频回声消除只能启用浏览器侧约束，实际效果必须现场验证。
- 多路高清长时间录制对 CPU、GPU、USB 控制器和磁盘写入都有压力，需要现场压测。
- 当前阶段不实现 HIS、FTP、会议呼叫、云台控制和手机直播，它们属于后续阶段或阶段 1 扩展验证项。
