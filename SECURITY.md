# 安全说明

## 一、适用范围

本仓库是手术示教软件 PoC，不是生产医疗器械软件，不应用于真实诊疗、手术决策或患者管理。

## 二、患者信息

1. 不要向仓库提交真实患者姓名、住院号、身份证号、影像、病历、音视频或任何可识别个人身份的信息。
2. 当前 HIS 数据为本地模拟数据，只用于验证录像元数据绑定流程。
3. 后续接入真实 HIS 前，必须补充鉴权、审计、脱敏、权限控制和数据最小化策略。

## 三、漏洞报告

如果发现安全问题，请不要在公开 issue 中披露可利用细节。请通过私有渠道联系仓库维护者，并提供：

1. 影响范围。
2. 复现步骤。
3. 预期风险。
4. 建议修复方向。

## 四、当前未实现的安全能力

1. 生产级终端鉴权和用户级权限控制；当前信令共享令牌只用于 PoC 门禁。
2. TLS 和证书管理。
3. 合规操作审计；当前内存事件日志仅用于 PoC 调试和验收追踪。
4. 患者信息访问控制。
5. 录像文件加密。
6. AI 处理结果合规审查。

这些能力必须在生产化阶段补齐，不能用当前 PoC 直接替代。
