# 阶段 2 呼叫与互动 PoC 测试计划

## 测试目标

阶段 2 用于验证手术室端与示教室端的互动流程是否可以进入详细设计，包括呼叫、模式确认、默认画面、多路拉取、布局调整、音频通话约束和参与人数上限。

## 当前实现边界

当前版本在同一 Electron 应用内模拟手术室端和示教室端控制流，并复用本地 4 路预览流作为远端可订阅画面。它不等同于真实跨终端媒体服务或 SFU。真实跨机器、跨网段和 Android 会议平板联调必须在后续阶段接入媒体服务后重新验证。

同时提供轻量 WebSocket 信令服务器，用于验证 C/S 控制平面。该服务器不转发音视频媒体，只验证终端注册、通讯录、呼叫、接受、拒绝、会话订阅和参与上限。

## 测试用例

| 编号 | 测试项 | 操作 | 通过标准 | 结果 |
|---|---|---|---|---|
| P2-01 | 示教室呼叫手术室 | 选择通讯录终端，选择交互或仅收看，点击示教室呼叫手术室。 | 出现待确认呼叫，显示发起方、接收方和请求模式。 |  |
| P2-02 | 手术室反向呼叫 | 点击手术室呼叫示教室。 | 出现待确认呼叫，方向正确。 |  |
| P2-03 | 模式确认 | 接收方选择确认交互或确认仅收看。 | 任一方选择仅收看时，最终会话为仅收看；双方交互时为交互模式。 |  |
| P2-04 | 默认画面 | 接受呼叫后建立连接。 | 默认订阅并显示通道 1。 |  |
| P2-05 | 多路拉取 | 勾选通道 2、通道 3 或通道 4。 | 远端显示区可按需增加对应画面，不强制推送全部通道。 |  |
| P2-06 | 布局调整 | 切换单画面、双画面、四画面。 | 远端显示区布局随选择变化。 |  |
| P2-07 | 音频通话 | 交互模式下点击建立音频通话。 | 可建立本地音频轨道，并启用回声消除、降噪、自动增益约束。 |  |
| P2-08 | 参与上限 | 设置上限为 2，模拟新增参会者。 | 超过上限时提示新用户被拒绝。 |  |
| P2-09 | 标注可见 | 输入标注内容并勾选标注可见。 | 远端显示区出现标注覆盖层。 |  |
| P2-10 | 信令注册 | 手术室端、示教室端和观摩端分别注册到信令服务器。 | 服务器返回注册成功，通讯录包含 3 个在线终端。 |  |
| P2-11 | 信令呼叫 | 示教室端通过信令服务器呼叫手术室端。 | 手术室端收到来电事件，呼叫 ID 一致。 |  |
| P2-12 | 信令会话 | 手术室端接受呼叫并设置参与上限。 | 双方收到会话开始事件，默认订阅通道 1。 |  |
| P2-13 | 信令订阅 | 示教室端订阅通道 1、通道 2、通道 3。 | 服务器更新订阅并向会话参与方广播。 |  |
| P2-14 | 信令上限 | 第三方尝试加入已满会话。 | 返回 participant_limit 错误。 |  |
| P2-15 | 前端信令接入 | 前端填写信令地址并注册本端，选择在线目标后发起信令呼叫。 | 远端模拟终端收到来电并接受后，前端进入已建立会话状态。 |  |
| P2-16 | 信令标注同步 | 前端在已建立信令会话中填写标注并开启远端可见。 | 远端模拟终端收到包含标注文本、可见状态和更新方 ID 的会话更新。 |  |
| P2-17 | 信令会话结束 | 已建立信令会话后点击结束连接。 | 远端模拟终端收到 session.ended，前端清理会话状态，服务端移除会话。 |  |
| P2-18 | 断连会话清理 | 已建立信令会话后关闭其中一个终端连接。 | 服务端移除会话，剩余参与方收到 reason 为 endpoint_disconnected 的 session.ended。 |  |
| P2-19 | 终端通道清单 | 终端注册时上报 channels 字段，前端选中在线信令目标。 | 通讯录返回目标终端绑定的通道元数据，前端可显示目标通道清单。 |  |
| P2-20 | 运行时配置 | 前端启动时读取 public/config.json。 | 信令地址和本端 ID 默认值与配置文件一致。 |  |
| P2-21 | 信令呼入接受 | 远端示教室终端通过信令呼叫前端注册的手术室端。 | 前端显示待确认呼叫，接受后双方进入最终模式正确的会话。 |  |
| P2-22 | 信令参会更新 | 已建立信令会话后，观摩端通过 session.join 加入。 | 前端参与数量和参与方列表随 session.updated 更新。 |  |
| P2-23 | 信令呼入拒绝 | 远端示教室终端通过信令呼叫前端注册的手术室端，前端点击拒绝。 | 远端收到 call.rejected，前端保持未建立会话状态。 |  |

## 自动化冒烟测试

```powershell
cd D:\CodeX\UST\phase1-poc
npm run test:smoke
npm run test:signaling
```

冒烟测试使用系统 Chrome 打开 Vite 页面，验证页面渲染、模拟预览、呼叫确认、多路拉取、布局切换、标注、参与上限提示，以及前端连接真实 WebSocket 信令服务器后进入已接受会话。

信令测试会启动本地 WebSocket 服务器并模拟多终端消息交互，验证 C/S 控制平面的最小协议可用性。
