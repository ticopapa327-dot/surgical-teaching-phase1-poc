# 阶段 3 患者绑定与录像元数据 PoC 测试计划

## 测试目标

阶段 3 最小 PoC 用于验证录像文件能否与患者信息绑定。当前只使用前端内置模拟 HIS 数据，不连接真实 HIS，不处理患者主索引、接口鉴权、脱敏策略和医院系统字段映射。

## 当前实现边界

当前版本支持输入 HIS ID，查询本地模拟患者数据，并将当前患者对象写入新建录像的元数据。录像索引会显示患者名称和 HIS ID。

真实 HIS 联网、患者信息权限控制、接口审计、录像归档和回放中的患者信息展示规则尚未实现。

## 测试用例

| 编号 | 测试项 | 操作 | 通过标准 | 结果 |
|---|---|---|---|---|
| P3-01 | 模拟 HIS 查询 | 输入 `HIS-001` 并点击模拟 HIS 查询。 | 显示患者、HIS ID、科室、术式和基本信息。 |  |
| P3-02 | 查询失败 | 输入不存在的 HIS ID 并查询。 | 提示未查询到患者，新录像不绑定患者信息。 |  |
| P3-03 | 录像患者绑定 | 查询到患者后启动并停止一次选中通道录制。 | 录像索引显示患者名称和 HIS ID。 |  |
| P3-04 | 清除绑定 | 查询到患者后点击清除绑定。 | 当前患者绑定被清除，后续录像不写入患者字段。 |  |

## 自动化覆盖

```powershell
npm run test:smoke
```

当前烟测已覆盖 `P3-01`、`P3-02`、`P3-03` 和 `P3-04`。
