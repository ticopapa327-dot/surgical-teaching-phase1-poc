# AI 处理接口预留说明

## 一、当前目标

当前 PoC 只验证录像文件可以被加入 AI 处理任务队列，为后续识别、质控、结构化摘要或关键帧分析服务预留任务入口。

当前版本不执行真实 AI 推理，不上传录像，不调用外部模型，也不生成医学结论。

## 二、当前任务字段

本地模拟 AI 任务包含：

| 字段 | 说明 |
|---|---|
| `id` | 本地任务 ID |
| `recordingId` | 录像 ID |
| `channelLabel` | 录像通道 |
| `patient` | 录像绑定的患者元数据，可为空 |
| `status` | 当前固定为 `queued` |
| `createdAt` | 入队时间 |

## 三、测试覆盖

当前前端烟测会执行以下流程：

1. 模拟 HIS 查询并绑定患者。
2. 完成一次短录制。
3. 选择录像记录。
4. 点击加入 AI 队列。
5. 验证 AI 队列数量更新为 1。

## 四、后续扩展边界

后续真实 AI 接入应至少补齐：

1. AI 服务地址和鉴权配置。
2. 录像文件上传或本地文件路径授权。
3. 任务状态轮询或回调。
4. 处理结果结构化存储。
5. 患者信息脱敏与权限控制。
6. 医学风险提示，避免将模型输出直接作为诊断结论。
