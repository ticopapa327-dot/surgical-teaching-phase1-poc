# 2026-06-10 自主开发阶段进展摘要

## 一、本轮开发结论

本轮在不扩展真实 HIS、SFU、Android 平板、手机直播等外部依赖的前提下，优先推进了可在本仓库自动验证的能力。当前主分支已经恢复稳定，最近提交均已推送到公开仓库并通过 GitHub Actions CI。

## 二、已完成能力

1. 修复待处理呼叫超时 UI 用例的 CI 竞态，将 50 ms 级服务端超时从 UI 渲染测试中解耦。
2. 增加示教室端或观摩端“请求重发媒体”能力，通过 `media-refresh-request` 让手术室端按当前订阅重新发布 WebRTC 媒体。
3. 增加远端音频输出设备选择，支持 Chromium `setSinkId`，不支持时回退系统默认输出。
4. 增加 Electron 录像 FTP 上传 PoC，通过 `UST_FTP_*` 环境变量配置服务器、账号、FTPS 和远端目录。
5. 增加 UVC PTZ 控制入口，支持浏览器可用的 `pan`、`tilt`、`zoom` 能力探测与控制；不支持时明确提示。
6. 增加录像索引搜索，可按文件名、通道、患者姓名、HIS ID、科室和术式过滤。
7. 增加 AI 队列明细展示，显示任务状态、录像通道、文件名、时长、大小和患者绑定信息。
8. 补强信令契约测试，覆盖 `media-refresh-request` 的转发和事件日志摘要，确保日志不记录原始媒体协商负载。
9. 修正阶段文档边界，将已实现的 PoC 能力与仍需后续专项的生产级能力区分开。

## 三、验证结果

1. 多次执行 `npm run verify`，覆盖构建、高危依赖审计、录制管理、前端烟测、患者绑定、Phase 2/3 信令 UI 和信令契约测试，均通过。
2. 对 Phase 3 WebRTC 媒体用例执行 `--repeat-each=2` 重复稳定性验证，6 个媒体用例共 12 次全部通过。
3. 每个功能提交均推送到 GitHub，并观察对应 GitHub Actions CI 通过。

## 五、仍需真实环境复测的边界

1. 真实 USB 4 路采集卡长时间运行、码率、丢帧、CPU/GPU/内存占用。
2. 真实双机音频端到端延迟、回声消除、扬声器回放和不同声卡路径。
3. UVC PTZ 在目标摄像机上的实际响应；非 UVC、VISCA、Pelco、串口或厂商 SDK 云台控制仍需专项接入。
4. FTP 上传的真实服务器连通性、权限、目录策略、断点续传、失败重试和归档审计。
5. 生产级 SFU、RTSP/SRT 输入、手机直播分发和 Android 会议平板端。
