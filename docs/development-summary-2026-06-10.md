# 2026-06-10 自主开发阶段进展摘要

## 一、本轮开发结论

本轮在不扩展真实 HIS、SFU、Android 平板、手机直播等外部依赖的前提下，优先推进了可在本仓库自动验证的能力。当前主分支已经恢复稳定，最近提交均已推送到公开仓库并通过 GitHub Actions CI。

## 二、已完成能力

1. 修复待处理呼叫超时 UI 用例的 CI 竞态，将 50 ms 级服务端超时从 UI 渲染测试中解耦。
2. 增加示教室端或观摩端“请求重发媒体”能力，通过 `media-refresh-request` 让手术室端按当前订阅重新发布 WebRTC 媒体。
3. 增加远端音频输出设备选择，支持 Chromium `setSinkId`，不支持时回退系统默认输出。
4. 增加 Electron 录像 FTP 上传 PoC，通过 `UST_FTP_*` 环境变量配置服务器、账号、FTPS 和远端目录。
5. 增加 UVC PTZ 控制入口，支持浏览器可用的 `pan`、`tilt`、`zoom` 能力探测与控制；不支持时明确提示。
6. 增加录像索引搜索，可按文件名、通道、患者姓名、HIS ID、科室和术式过滤。
7. 增加 AI 队列明细展示，显示任务状态、录像通道、文件名、时长、大小和患者绑定信息。
8. 补强信令契约测试，覆盖 `media-refresh-request` 的转发和事件日志摘要，确保日志不记录原始媒体协商负载。
9. 修正阶段文档边界，将已实现的 PoC 能力与仍需后续专项的生产级能力区分开。

## 三、验证结果

1. 多次执行 `npm run verify`，覆盖构建、高危依赖审计、录制管理、前端烟测、患者绑定、Phase 2/3 信令 UI 和信令契约测试，均通过。
2. 对 Phase 3 WebRTC 媒体用例执行 `--repeat-each=2` 重复稳定性验证，6 个媒体用例共 12 次全部通过。
3. 每个功能提交均推送到 GitHub，并观察对应 GitHub Actions CI 通过。

## 五、续跑新增成果

1. 修复三端会议中接收端点击“停止媒体链路”误伤其他接收端的问题；非 owner 终端停止媒体时只通知手术室 owner 清理本端对应 PeerConnection。
2. 增加停止后“请求重发媒体”的三端自动化覆盖，验证观摩端停止自身媒体后可恢复，且示教室端持续接收不受影响。
3. 扩展前端信令事件日志显示，增加 from/to、信号类型、SDP 类型、通道和轨道数摘要，并将前端显示范围从最近 5 条扩展为最近 10 条。
4. 增加局域网启动脚本测试，覆盖 `dev:lan:check` 成功路径和端口占用失败提示；`npm run verify` 已纳入脚本级测试。
5. 增加音频诊断状态，显示安全上下文、采集能力、输入/输出设备数量、输出选择能力、当前麦克风和回放设备。
6. 在 PeerConnection 连接诊断中显示本地发送音频轨道数和远端接收音频轨道数，便于区分“视频通、音频未入链路”。
7. 扩展 `/health` 响应，增加 `startedAt`、`uptimeSeconds`、`eventLogSize` 和 `eventLogLimit`，前端健康检查同步显示运行时间和事件日志容量。
8. 加固 Electron 录像文件路径，定位、导出、删除、FTP 上传和 `recording://` 读取均限制在录像目录内，避免本地索引被篡改后访问目录外文件。
9. Electron 录像写入改为等待底层 stream 回调，并覆盖写入成功和失败场景，避免磁盘写入错误被误报为成功。
10. 媒体统计增加 ICE 候选类型路由显示，用于区分局域网直连、TURN/relay 或未建立候选对等情况。
11. 增加“复制诊断快照”，输出本端控制面、音频、媒体、PeerConnection、最近信令事件和通道状态摘要，排除信令令牌、设备 ID、患者信息、SDP、ICE candidate 原文和媒体数据。
12. 诊断快照纳入每路通道源模式、预览状态、live 视频轨道数、录制状态和已选设备显示标签，便于排查 USB 采集卡不可见、选错源或未预览。
13. 已连接信令时，复制诊断快照会先自动刷新 `/events`，减少现场忘记点击“刷新事件”导致 recentEvents 为空的问题。
14. 服务端事件日志增加 `session.subscription.updated` 和 `session.annotation.updated` 摘要事件，并在契约测试中验证标注正文不会进入事件日志。
15. `/events` 支持 `limit` 查询参数，诊断快照只读取最近 10 条事件，降低事件日志扩大后的前端取数开销。
16. 局域网启动脚本支持 `UST_PREFERRED_ADAPTER`，在多网卡机器上可优先推荐有线网卡地址，默认绑定策略不变。
17. 增加 `npm-audit-retry` 脚本测试，模拟审计接口瞬断后重试成功，避免高危依赖审计结果产生歧义。
18. 信令服务停止时主动终止所有 WebSocket 客户端，并增加契约测试，降低测试失败或本地退出时的挂起风险。
19. 局域网启动脚本测试覆盖 `--smoke` 模式，验证实际启动信令服务和 Vite 前端后可自动退出。
20. README 已同步最新诊断快照、事件限量读取、首选网卡和脚本测试说明，便于开源使用者从入口文档理解当前能力。
21. README 增加 GitHub Actions CI 状态徽章，公开仓库入口可直接看到 main 分支验证状态。
22. 信令事件日志增加 `call.accepted` 摘要，并在契约测试和前端快照用例中覆盖，便于现场区分“已接受但未发媒体”和“呼叫未完成”。
23. 增加 `test:media` 与 `test:media:repeat` 脚本，分别用于快速运行 Phase 3 WebRTC 媒体组和执行 `--repeat-each=2` 稳定性复测。
24. 增加仅收看模式媒体回归断言，验证三端差异订阅下不会附带本地或远端音频轨道，避免仅收看场景误建音频链路。
25. 媒体统计拆分为视频发送/接收码率和包发/收/丢，便于区分发布端未发送、接收端未收到或网络路径异常。
26. 诊断快照增加 `media.statsMetrics` 结构化 WebRTC 指标，覆盖有媒体和无媒体两种自动化用例，便于后续现场快照汇总。
27. 增加 `public/config.example.json`，提供局域网信令地址和 STUN/TURN 占位配置，并在 README 和两台电脑手册中强调不得提交真实凭据。
28. 增加运行时配置示例测试，将示例 JSON、信令 URL、终端角色和 STUN/TURN 占位项纳入 `test:scripts`。
29. 增加诊断快照 CSV 汇总脚本，可把两端保存的 JSON 快照转换为视频码率、包计数、RTT 和 ICE 路由表格。
30. 加固录像文件名清理逻辑，避免 `.` / `..` 这类路径语义名称进入 FTP 远端文件名。
31. 信令 WebSocket 增加单消息 payload 上限，默认 1 MiB，并在 `/health` 暴露 `maxPayloadBytes`，契约测试覆盖超大消息关闭。
32. 会话状态区显示 `mediaRoomId`，便于现场确认当前信令会话对应的媒体房间。
33. `peer.signal` 透传消息、事件日志、诊断快照和协议文档均纳入 `mediaRoomId`；服务端会拒绝与会话不匹配的媒体房间 ID，降低后续 SFU 房间映射或 stale session 排障风险。

## 六、仍需真实环境复测的边界

1. 真实 USB 4 路采集卡长时间运行、码率、丢帧、CPU/GPU/内存占用。
2. 真实双机音频端到端延迟、回声消除、扬声器回放和不同声卡路径。
3. UVC PTZ 在目标摄像机上的实际响应；非 UVC、VISCA、Pelco、串口或厂商 SDK 云台控制仍需专项接入。
4. FTP 上传的真实服务器连通性、权限、目录策略、断点续传、失败重试和归档审计。
5. 生产级 SFU、RTSP/SRT 输入、手机直播分发和 Android 会议平板端。
